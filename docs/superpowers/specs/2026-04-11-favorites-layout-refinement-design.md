# SlimGPT 收藏与 PDF 布局修正设计

## 目标
修正上一版收藏功能和 PDF 按钮的视觉位置错误，保证它们不遮挡正文、不挤占聊天视野，同时保持入口稳定、易找、易点。

## 变更范围
- 调整 PDF 按钮位置与样式。
- 调整消息区用户问题星标的位置与配色。
- 保留问题栏中的星标入口，但同步更新星标配色。
- 不改收藏数据模型，不改持久化逻辑，不增加新筛选能力。

## PDF 按钮设计
- PDF 按钮不再贴在输入区域附近。
- 按钮改为“左侧导航栏正下方、单独悬浮”。
- 锚点不是侧栏宽度的近似值，而是左侧导航栏底部区域本身，目标位置是原来底部标签区域正下方的空白区。
- 按钮与导航栏底部保持很小间距，视觉上像导航底部延伸出来的独立工具钮。
- 按钮使用更显眼的黑色高对比底色，维持短标签 `PDF`，便于快速识别。
- 按钮位置固定，不跟随聊天正文或会话项漂移。

## 消息区星标设计
- 星标始终显示，不依赖 hover。
- 星标不再覆盖问题文字。
- 星标定位改为整条用户提问气泡外侧的右边缘，和气泡之间只保留很小间距，视觉上是“紧挨着对话框”。
- 星标不放在气泡内部，不占正文宽度，不盖住任何文字。
- 垂直位置以对话框中线附近为主，不做右上角角标式摆放。
- 星标尺寸维持小按钮级别，但颜色更醒目：
  - 未收藏：更亮的暖灰/浅金空心星；
  - 已收藏：高饱和黄色实心星。
- 星标按钮仍只出现在用户问题消息上，不出现在 assistant 回复上。

## 问题栏设计
- 问题栏结构保持现有 `Qx + 星标 + 正文摘要` 布局，不再改位置。
- 星标配色与消息区保持一致：未收藏为空心亮色星，已收藏为黄色实心星。
- 点击问题栏星标仍只切换收藏，不触发跳转。

## 交互要求
- 点击消息区星标与问题栏星标，状态必须即时双向同步。
- 星标点击区域需要明确，不应误触正文或跳转行为。
- PDF 按钮始终可见，但不应压在聊天输入框、消息气泡或右上问题栏之上。

## 实现思路
- `content.js`
  - 修改 PDF 悬浮按钮的锚点计算与插入位置，直接依附左侧导航栏底部区域。
  - 调整消息区星标的挂载宿主和定位策略，确保按钮挂在整条用户消息容器外侧右边缘，而不是正文文字内部。
- `styles.css`
  - 调整 PDF 按钮的定位、颜色和层级。
  - 给用户消息容器增加外侧定位空间，让星标能贴边但不压字。
  - 提升星标的默认可见度和激活态颜色。

## 风险与边界
- ChatGPT 用户消息卡片的 DOM 结构可能比当前截图更复杂，消息区星标的宿主选择需要尽量取“整条用户消息容器”而不是内部文本块。
- 左侧导航栏宽度在不同窗口尺寸下可能变化，PDF 按钮的固定定位要兼顾窄屏，避免跑出视口。
- 这次不重新设计问题栏整体布局，只修正颜色和一致性。

## 验证
- `node --check content.js`
- 现有 Node 测试继续通过
- 浏览器手工验证：
  - PDF 按钮出现在左侧导航栏底部标签区域正下方，单独悬浮；
  - 消息区星标紧挨用户问题对话框右侧，但不覆盖文字；
  - 星标始终显示且颜色更醒目；
  - 消息区与问题栏星标状态同步；
  - 不同窗口宽度下按钮不遮挡主要内容。
