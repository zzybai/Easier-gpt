# SlimGPT 问题收藏设计

## 目标
在 SlimGPT 的用户问题体验中加入“收藏问题”能力，让用户可以在消息区和右上角问题栏两处标记重要问题，并在刷新后保留收藏状态。

## 范围
- 只允许收藏用户问题，不处理 assistant 回复。
- 收藏入口同时存在于消息区用户提问卡片和右上角问题栏。
- 消息区与问题栏共用同一份收藏状态，任一处切换都要立即同步另一处。
- 收藏状态需要在页面刷新后保留。
- 问题栏维持现有顺序和跳转行为，不因为收藏而重排。

## 非目标
- 本次不增加“只看收藏”筛选。
- 本次不改导出、打印或 PDF 格式。
- 本次不为 assistant 消息增加收藏或评分能力。

## 体验要求
- 星标要低干扰，不影响正常阅读视野。
- 消息区星标放在用户消息块右上角，尺寸较小，默认使用低对比度空心星。
- 收藏后显示黄色实心星，但不放大、不新增额外提示文案。
- 问题栏的 `Qx` 下方显示星标，作为第二个直接操作入口。
- 点击问题栏星标只切换收藏，不触发跳转；点击问题项其他区域仍保持跳转。

## 数据模型
- 收藏对象按“当前会话 + `turnIndex` + 归一化后的用户问题文本”生成稳定指纹。
- 不使用运行时生成的 `data-slimgpt-id` 作为持久化键，因为该值刷新后不稳定。
- 归一化文本沿用现有问题栏文本清洗逻辑，至少包含空白折叠和 trim，避免轻微 DOM 变化导致收藏失效。
- 收藏状态存入 `chrome.storage.local`。
- 存储结构按会话维度隔离，避免不同会话间串收藏数据。

## 架构
- 新增一个纯函数模块，负责基于会话路径、`turnIndex` 和问题文本构建收藏键，便于 Node 测试。
- `content.js` 负责：
  - 从 `STATE.messages` 中识别用户问题；
  - 读写 `chrome.storage.local`；
  - 在消息区用户消息上挂载星标按钮；
  - 在问题栏项中渲染星标并处理点击；
  - 维护两处 UI 的同步。
- `styles.css` 负责星标按钮和问题栏星标的轻量视觉样式。
- `manifest.json` 需要把新纯函数模块加入 content script 列表。

## 交互细节
- 消息区用户问题星标：
  - 初次渲染时插入到用户消息卡片右上角。
  - 未收藏显示为空心星，颜色中性偏灰。
  - 收藏后显示黄色实心星。
  - 点击只切换收藏，不影响正文选择和其他消息交互。
- 问题栏星标：
  - 位于 `Qx` 下方居中。
  - 未收藏显示空心星，已收藏显示黄色实心星。
  - 星标始终可见，保证它是明确的操作入口。
  - 星标按钮与问题项主按钮分离，避免误触跳转。

## 同步与恢复
- 页面初始化或会话切换时，从本地存储读取当前会话的收藏集合。
- 新问题进入 `STATE.messages` 后，若能生成与已存收藏匹配的键，应自动显示为已收藏。
- 在任一入口切换收藏后，立即刷新消息区和问题栏显示，并异步写回本地存储。
- 读取或写入本地存储失败时，只记录控制台错误，不阻断聊天页面正常使用。

## 兼容性与风险
- ChatGPT 页面可能持续重渲染，因此消息区星标插入逻辑必须可重复执行且避免重复挂载。
- 折叠/占位消息不应错误显示收藏按钮；只有真实用户消息卡片需要按钮。
- 如果用户显著修改了同一轮问题文本，旧收藏可能无法自动命中；这是可接受的稳定性边界。

## 验证
- Node `--test` 覆盖收藏键构建逻辑和问题栏收藏展示数据。
- `node --check content.js` 验证脚本语法。
- 浏览器手工验证：
  - 消息区和问题栏的星标显示正确；
  - 双入口点击后状态同步；
  - 问题栏星标点击不触发跳转；
  - 刷新页面后收藏状态恢复；
  - 新问题持续出现时，新旧问题星标都保持正常。
