# Easier-GPT README 双语文档设计

## 目标
为仓库补充中英文两套等价 README，并让 GitHub 仓库首页默认展示中文版本，同时为英文读者提供清晰、直接的语言切换入口。

## 范围
- 保留根目录 `README.md` 作为中文默认首页。
- 新增根目录 `README.en.md` 作为英文版本。
- 两个 README 的章节结构、信息范围与限制说明保持等价。
- 中英文 README 顶部都提供语言切换链接。
- 本次不修改功能代码、不引入自动语言检测、不调整仓库目录结构。

## 文档结构
- `README.md` 继续使用当前中文内容，保留已经删减后的结构：项目简介、创新亮点、功能概览、安装、使用方式、隐私与安全、许可与商用提醒、项目结构、适用场景。
- `README.en.md` 对上述内容做完整等价翻译，不新增额外卖点或删减限制说明。
- 两个文件顶部使用纯 Markdown 文本链接，避免依赖图片按钮或 GitHub 不支持的交互组件。

## GitHub 展示策略
- 由于 GitHub 仓库首页默认读取根目录 `README.md`，因此中文保持在该文件中。
- 英文文档通过顶部链接进入，不额外创建跳转页。
- 中文页顶部显示 `中文 | [English](./README.en.md)`。
- 英文页顶部显示 `[中文](./README.md) | English`。

## 一致性规则
- 任一 README 的章节增删、表述修订或许可提醒更新时，另一语言版本必须同步修改。
- 英文文档以忠实翻译为原则，不对中文原文做功能性扩写或营销性重写。
- 若后续 README 继续调整，以中文 `README.md` 为主文档，英文版同步跟进。

## 验证
- 检查 `README.md` 首页顶部语言切换链接能正确跳转到 `README.en.md`。
- 检查 `README.en.md` 顶部语言切换链接能正确跳回 `README.md`。
- 检查两份 README 的章节顺序和限制说明是否一一对应。
- 检查 GitHub 渲染下链接、列表与代码块格式是否正常。
