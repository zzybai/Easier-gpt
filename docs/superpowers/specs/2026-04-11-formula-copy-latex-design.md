# SlimGPT 公式点击复制 LaTeX 设计

## 目标
让用户在 ChatGPT 页面中直接点击显示出来的数学公式本体，一键复制该公式对应的 LaTeX 源码。

## 范围
- 支持块级公式复制 LaTeX。
- 支持行内公式复制 LaTeX。
- 点击公式本体即可复制，不新增独立按钮。
- 复制反馈通过轻量 hover/点击提示完成。

## 非目标
- 本次不支持 MathML / Word 格式复制。
- 本次不增加格式切换菜单或扩展设置项。
- 本次不处理没有稳定 LaTeX 源注解的非 KaTeX 数学节点。

## 数据来源
- 公式 LaTeX 源优先来自 `annotation[encoding="application/x-tex"]`。
- 只有存在该注解的公式节点才具备点击复制能力。
- 不从渲染后的可视文本反推公式源码，避免错误复制。

## 交互
- 鼠标悬停到公式本体时，显示轻量提示，告诉用户这里可复制 LaTeX。
- 点击公式后，直接复制 LaTeX 源码到剪贴板。
- 复制成功后，提示短暂变为 `Copied`。
- 行内公式提示比块级公式更克制，避免影响正文阅读。

## 技术方案
- 复用现有公式渲染链路中的 KaTeX 节点识别能力。
- 在公式渲染完成后，为对应容器挂载：
  - `data-slimgpt-latex-copy="1"`
  - `data-slimgpt-latex-source="<latex>"`
  - 统一的 hover/复制反馈属性
- 通过一个全局点击委托处理复制，避免给每个公式反复绑定独立监听器。
- 对动态渲染的新公式继续在 MutationObserver 中补齐复制能力。

## 容器策略
- 块级公式：以 `.katex-display` 或其可见公式容器为点击目标。
- 行内公式：以 `.katex` 外层包裹节点为点击目标。
- 同一个公式只挂一层复制目标，避免重复嵌套导致事件异常。

## 风险与边界
- 如果 ChatGPT 页面后续公式结构变化，只有仍然保留 `application/x-tex` 注解的公式能继续工作。
- 行内公式点击复制可能与用户文本选择产生轻微冲突，因此提示要明确且点击区域仅限公式本体。
- 复制失败时不阻断页面交互，只是不显示成功状态。

## 验证
- Node 侧补充公式复制目标识别/绑定的最小测试。
- `node --check content.js`
- 浏览器手工验证：
  - 块级公式点击后能复制 LaTeX；
  - 行内公式点击后能复制 LaTeX；
  - hover 时提示清晰但不碍眼；
  - 复制成功后出现短暂反馈。
